隨身寫作＋改稿夥伴

打開就能用的對話介面，適合「單篇、當下」的任務。

  • 改稿濾鏡：貼一段稿，請它抓 AI 腔、贅字，順便調整整體語氣。
  • 長文改寫：千字以上的文章，也能維持前後語氣一致。
  • 關鍵字分群＋搜尋意圖：丟一串關鍵字，請它依搜尋意圖分類，再整理出文章架構。
  • E-E-A-T 體檢：請它從「經驗、專業、權威、可信」角度檢查內容是否完整。

不用寫程式的桌面 AI 助手

Claude 的桌面版，能直接讀電腦裡的資料夾、操作檔案、產出成品，全程用中文操作，不需要寫程式。

Anthropic 官方把它定位成「給知識工作者的 Claude Code」。
  • 產架構（大綱）：丟目標關鍵字＋10+ 篇競品連結，請它產出帶有 H 標籤的架構與每段要點。
  • 抓用戶需求（搜尋意圖）：貼上 SERP 前幾名或 People Also Ask，讓它整理「讀者真正想知道什麼、文章該回答哪些問題」。
  • 出文章初稿：架構確認後，把參考資料丟給它，就能產出符合團隊語氣的初稿，再由編輯精修。
  • 定時任務：例如每週整理 AI 趨勢、每月提醒發布下一集，都可以排程自動完成。
  • 批次整理：指定一個資料夾，就能批次改檔名、分類、去重、整理重點。

把整個內容庫的重複流程自動化

雖然名字叫 Code，但目前（2026）已進化成「非工程師也能用的自動化執行層」。Search Engine Land 稱它為「SEO 指揮中心」，Semrush 則定義成「SEO 分析師」。

案例：Ahrefs 內容總監 Ryan Law 分享，他用 23 個 Skill 串成 Claude Code Pipeline，只要 6～12 分鐘就能完成一篇可發布的 SEO 初稿。
  • 批次產架構＋用戶需求：一次處理一批主題，直接結合 SERP 與 GSC 數據整理搜尋意圖，再依同一份 Brief 模板批次產出標準化架構。
  • 可重跑的產稿流程：把「研究 → 用戶需求 → 架構 → 初稿 → 稽核」串成一條 Pipeline，每次都能用同一套流程完成。
  • 全站內容稽核：一次掃描整個 Sitemap（上百個 URL），找出過時內容、內容缺口、內鏈問題、重複內容與品牌語氣不一致，最後整理出更新優先順序。
  • 結合真實數據做決策：串接 Google Search Console、Ahrefs 或 Semrush 匯出的資料，讓選題與改稿建議都有數據依據。
  • 定期自動報表：每月自動整理 GSC 成效報表，以及 AI 引用能見度追蹤。

❓ 架構、用戶需求、文章初稿，也能自動產嗎？

能，而且是編輯最有感的部分。差別在「規模」——同樣三件事，Cowork 適合「這一篇」，Code 適合「整批主題」。

要產的東西Cowork（一次一篇）Code（一次一批／一整站）
架構（大綱）丟關鍵字＋競品連結，產出這一篇的 H2／H3 與每段要點對一批主題套同一份 Brief 模板，批次產、架構標準一致
用戶需求（意圖）貼 SERP 前幾名或 People Also Ask，讓它歸納讀者想知道什麼直接接 GSC／SERP 數據，意圖分析有數據撐腰、不靠猜
文章初稿架構定稿＋參考資料丟給它，一篇一篇出團隊語氣的初稿把「研究 → 需求 → 架構 → 初稿 → 稽核」串成 pipeline，一鍵跑、品質一致
⚠️ 提醒：AI 負責出初稿與架構，事實查證、E-E-A-T、法規用字仍要編輯把關。工具加速產出，判斷還是人做。

💡 補充：什麼是「Skill」？

我們正在導入的核心概念。

Ahrefs 的解釋很直白：Skill 就像一份可以重複使用的工作 SOP，把你每次都要交代的規則（語氣、格式、檢查清單）寫成一份 SKILL.md。之後只要用自然語言描述需求，Claude 就會自動套用，不需要每次重新貼 Prompt。

建立 Skill 也不用寫程式，透過官方 Skill Creator 就能一步步完成。除了我們編輯團隊正在不斷優化的 Skill，Claude Code 也內建不少常用 Skill：

/batch/debug/loop/simplify

  • 把內容簡報改寫成特定語氣
  • 用固定 QA 清單檢查一篇草稿
